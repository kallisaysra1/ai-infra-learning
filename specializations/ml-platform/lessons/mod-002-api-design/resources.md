# Resources — API Design

## Reading
- [Google API Design Guide](https://cloud.google.com/apis/design)
- [Microsoft REST API Guidelines](https://github.com/microsoft/api-guidelines)
- [API Stylebook](http://apistylebook.com/) — comparison of major API style guides
- *Designing Web APIs* — Brenda Jin et al. (O'Reilly)
- *Continuous API Management* — Mehdi Medjaoui (O'Reilly)

## Specifications
- [OpenAPI 3.1 spec](https://spec.openapis.org/oas/v3.1.0)
- [AsyncAPI 2.6](https://www.asyncapi.com/docs/reference/specification/v2.6.0)
- [JSON:API](https://jsonapi.org/)
- [GraphQL spec](https://spec.graphql.org/)

## Tools
- [OpenAPI Generator](https://openapi-generator.tech/) — codegen for 50+ langs
- [Schemathesis](https://schemathesis.readthedocs.io/) — property-based contract tests
- [Prism](https://stoplight.io/open-source/prism) — mock server from OpenAPI
- [Spectral](https://stoplight.io/open-source/spectral) — OpenAPI linter
