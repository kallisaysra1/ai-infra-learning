# Deliverables — Thought Leadership Portfolio

This directory is where your portfolio lands. The full project requires 10 deliverables (D1–D10). Use the conventions below so a reviewer (peer who has run a similar portfolio, sponsor) can navigate without your help.

Note: this portfolio is unusual — several deliverables are **external artifacts** (published paper, accepted CFP, merged PR, signed advisory agreement) rather than internal documents. Link the externals from your in-directory files.

## Deliverable checklist

Use when self-assessing against `rubric.md`.

### D1 — Strategy document
- [ ] 15–25 pages
- [ ] Covers thesis + 5 channels + operating model + measurement + ethics
- [ ] Maps each artifact to ≥ 1 requirement ID
- [ ] Peer-reviewed; notes captured

### D2 — Thesis
- [ ] Single sentence (≤ 25 words) + 2–3 paragraph defense
- [ ] ≥ 3 concrete supporting examples
- [ ] Anti-thesis named with response
- [ ] Peer recitation test passed

### D3 — Conference talk
- [ ] Venue chosen with reasoning
- [ ] CFP submitted (with confirmation link)
- [ ] Deck 30–45 slides drafted
- [ ] Rehearsed at least once
- [ ] Outcome captured (accepted / rejected / pending)
- [ ] If rejected: revised version + backup venue identified

### D4 — Paper / essay
- [ ] Venue chosen with reasoning
- [ ] ≥ 3,500 words
- [ ] Original evidence included
- [ ] Peer-edited
- [ ] Published OR at editor's desk with timeline

### D5 — OSS contribution
- [ ] Project chosen with thesis alignment + maintainer culture check
- [ ] 6-month contribution arc documented
- [ ] First contribution merged OR in active review with maintainer interaction
- [ ] Second contribution queued with specific issue
- [ ] Project CONTRIBUTING / RFC process followed

### D6 — Advisory engagements
- [ ] 5–10 targets identified
- [ ] ≥ 3 active conversations
- [ ] ≥ 1 signed OR term sheet in negotiation
- [ ] Employer COI cleared per process
- [ ] Time-boxing documented (≤ 8h/month/engagement)
- [ ] Deal structure templates archived

### D7 — Content calendar
- [ ] 24+ topics queued (12 months bi-weekly)
- [ ] ≥ 4 buffer pieces drafted
- [ ] Platform live (newsletter + personal site + RSS)
- [ ] First piece published with link
- [ ] Cadence on personal calendar
- [ ] Topics ladder back to thesis

### D8 — Employer alignment
- [ ] Sponsor signoff captured (in writing — email or signed memo)
- [ ] IP boundaries documented and acknowledged
- [ ] Approval workflow live (named contact, SLA)
- [ ] Quarterly sponsor update template

### D9 — Leverage metrics dashboard
- [ ] Per-channel metrics defined with collection mechanism
- [ ] Monthly review on calendar
- [ ] Q1 baseline captured
- [ ] Reallocation triggers identified

### D10 — Reflection memo
- [ ] Written at end of cycle (4–8 pages)
- [ ] What worked / what didn't, per channel
- [ ] Channel reallocation for cycle 2
- [ ] 12-month outcomes vs. goals — honest comparison
- [ ] Sponsor update sent
- [ ] Next-cycle plan committed

## File naming conventions

- Buffer pieces and published pieces: `YYYY-MM-DD-short-kebab-title.md`
- External links: capture in a `.md` file with link + retrieved-date + screenshot (for archival when external content disappears)
- Decks: PDF + source format; speaker notes in sibling `.md`
- CFP submission: full text + venue link + submission confirmation in `cfp-submission.md`

## Submission

When complete:
1. Run the self-assessment in `rubric.md`
2. File revisions; capture in `d01-strategy/peer-review-notes.md`
3. Request peer review from 2 personas:
   - A senior who has run a similar portfolio (the most important review)
   - Your sponsor (for employer alignment)
4. The reflection memo (D10) is written **at the end of cycle 1 (month 12)**, not at submission time. You will return to this directory and add D10 once the cycle plays out.

This is the only project in the curriculum where the portfolio at submission is not complete — by design, this project is the foundation; the value compounds over the 12 months that follow. Treat the submission as a milestone, not a finish line.
