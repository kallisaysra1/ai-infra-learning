# Module 206: Advanced MLOps - Recommended Reading

## Books

### Essential Reading

**"Designing Machine Learning Systems" by Chip Huyen**
- Comprehensive guide to production ML systems
- Covers data engineering, model deployment, monitoring
- Real-world case studies and best practices
- **Focus**: Chapters 4-8 (Training, Deployment, Monitoring)

**"Machine Learning Design Patterns" by Valliappa Lakshmanan, Sara Robinson, Michael Munn**
- 30+ reusable patterns for ML systems
- Google's production ML practices
- Covers training, serving, and workflow patterns
- **Focus**: Patterns for serving and workflow orchestration

**"Building Machine Learning Powered Applications" by Emmanuel Ameisen**
- End-to-end ML product development
- From prototype to production
- Practical advice on iterations and monitoring
- **Focus**: Chapters on deployment and monitoring

**"Reliable Machine Learning" by Todd Underwood, Cathy Chen, Niall Murphy**
- O'Reilly book on ML reliability
- SRE principles for ML systems
- Incident management and debugging
- **Focus**: Chapters 5-7 (Operations, Monitoring, Incidents)

### Advanced Reading

**"Machine Learning Systems" by Chip Huyen (Stanford CS 329S)**
- University course materials (free online)
- Covers full ML lifecycle
- Industry best practices
- **Link**: https://stanford-cs329s.github.io/

**"MLOps: Continuous delivery and automation pipelines" by Google**
- Google Cloud MLOps guide
- Maturity model framework
- CI/CD for ML systems
- **Link**: https://cloud.google.com/architecture/mlops-continuous-delivery-and-automation-pipelines-in-machine-learning

**"Feature Store for Machine Learning" by Tecton**
- Comprehensive feature store guide
- Architecture patterns
- Real-world implementations
- **Link**: https://www.featurestore.org/

## Research Papers

### Feature Stores

**"Towards ML Engineering: A Brief History Of TensorFlow Extended (TFX)"**
- Google's production ML platform
- Feature engineering at scale
- Paper: https://arxiv.org/abs/2010.02013

**"Michelangelo: Machine Learning Platform at Uber"**
- End-to-end ML platform
- Feature store and model serving
- Link: https://eng.uber.com/michelangelo-machine-learning-platform/

**"Zipline: Airbnb's Machine Learning Data Management Platform"**
- Feature engineering infrastructure
- Blog post: https://databricks.com/session_na20/zipline-airbnbs-machine-learning-data-management-platform

### Model Deployment & Serving

**"TensorFlow Serving: Flexible, High-Performance ML Serving"**
- Production model serving system
- Paper: https://arxiv.org/abs/1712.06139

**"Clipper: A Low-Latency Online Prediction Serving System"**
- UC Berkeley's prediction serving system
- Model selection and caching strategies
- Paper: https://arxiv.org/abs/1612.03079

**"D3M: End-to-End Machine Learning Platform"**
- AutoML and automated pipelines
- Paper: https://arxiv.org/abs/1811.01389

### A/B Testing & Experimentation

**"Trustworthy Online Controlled Experiments: A Practical Guide to A/B Testing"**
- Comprehensive A/B testing guide
- Statistical methods and pitfalls
- Book by Kohavi, Tang, and Xu

**"Multi-Armed Bandits" by Kuleshov & Precup**
- Bayesian approaches to experimentation
- Thompson Sampling algorithms
- Paper: https://arxiv.org/abs/1707.02038

**"Large-scale Online Experimentation at Microsoft"**
- Enterprise A/B testing platform
- Statistical challenges at scale
- KDD paper

### MLOps & Production Systems

**"Hidden Technical Debt in Machine Learning Systems"**
- Classic paper on ML system complexity
- Technical debt patterns
- Paper: https://papers.nips.cc/paper/2015/file/86df7dcfd896fcaf2674f757a2463eba-Paper.pdf

**"Machine Learning: The High-Interest Credit Card of Technical Debt"**
- Google's perspective on ML technical debt
- Maintenance challenges
- Must-read for ML engineers

**"MLOps: Continuous Delivery for Machine Learning on Google Cloud"**
- Practical MLOps guide
- CI/CD patterns for ML
- Whitepaper: https://cloud.google.com/resources/mlops-whitepaper
